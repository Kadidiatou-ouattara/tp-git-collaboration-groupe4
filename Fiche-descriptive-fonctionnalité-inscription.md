&nbsp;Fiche Descriptive : Inscription des donneurs volontaires avec groupe sanguin et coordonnées



I - Mon Profil



Nom : Mohamed coulibaly/ flash



Ma mission : Créer la porte d'entrée pour les citoyens qui souhaitent sauver des vies en devenant donneurs volontaires.



II - Ma Tâche spécifique

Je dois concevoir et coder le système d'Inscription des donneurs volontaires.



Qu'est-ce que cela signifie concrètement ?

Je vais créer une interface où une personne peut s'enregistrer pour dire : "Je suis volontaire, voici mon groupe sanguin et voici comment me joindre si besoin."



III - Ce que je vais développer (Fonctionnalités)

Pour réussir cette tâche, je vais coder les éléments suivants :



Le Formulaire d'inscription :



Un champ pour le Nom et Prénom.



Un champ pour le Numéro de téléphone (crucial pour les alertes rapides).



Une liste déroulante pour choisir le Groupe Sanguin (A+, A−, B+, B−, AB+, AB−, O+, O−).



Un champ pour la Localisation (ville ou quartier) pour que l'hôpital le plus proche puisse trouver le donneur.



La Validation des données :



Vérifier que le numéro de téléphone est valide.



S'assurer que le groupe sanguin a bien été sélectionné.



Le Stockage :



Faire en sorte que ces informations soient enregistrées dans la base de données du projet pour être réutilisables par les hôpitaux en cas d'urgence.



IV - Les objectifs de ma tâche

Faciliter l'engagement : Créer un formulaire simple et rapide pour ne pas décourager les donneurs.



Fiabilité des données : Garantir que les coordonnées sont correctes pour éviter de perdre du temps en cas d'urgence vitale.



Disponibilité : Permettre aux citoyens de s'inscrire à tout moment, même s'il n'y a pas d'urgence actuelle.



V - Comment je vais procéder 

Étape 1 : Je crée ma branche Git ( feature-inscription-donneur).



Étape 2 : Je rédige cette fiche descriptive dans mon fichier de présentation.



Étape 3 : Je code la structure du formulaire (en HTML).



Étape 4 : Je mets en forme le formulaire pour qu'il soit agréable à utiliser (en CSS).



Étape 5 : Je teste mon code et je fais un commit pour sauvegarder mon travail.



Étape 6 : Je fais une Pull Request pour que le Lead Projet valide mon code.
